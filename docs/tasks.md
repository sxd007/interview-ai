### 任务 2: 错误处理完善

**目标**: 所有API返回标准化错误响应

**子任务**:

- [ ] **2.1 统一异常类型定义**
  - 优先级: P0
  - 预估工时: 1天
  - 文件位置: `src/core/exceptions.py`
  - 实现内容:
    - 定义基础异常类 `InterviewAIException`
    - 定义具体异常类型:
      - `ModelLoadError` - 模型加载失败
      - `VideoProcessingError` - 视频处理失败
      - `AudioProcessingError` - 音频处理失败
      - `GPUError` - GPU相关错误
      - `ValidationError` - 数据验证错误
    - 每个异常包含错误码、消息、详情
  - 验收标准:
    - 异常类型覆盖所有错误场景
    - 每个异常有唯一错误码
    - 包含详细错误信息

- [ ] **2.2 API错误处理中间件**
  - 优先级: P0
  - 预估工时: 1天
  - 文件位置: `src/api/middleware/error_handler.py`
  - 实现内容:
    - 全局异常捕获中间件
    - 标准化错误响应格式:
      ```json
      {
        "error_code": "MODEL_LOAD_ERROR",
        "message": "Failed to load emotion model",
        "details": {...},
        "timestamp": "2026-04-02T10:00:00Z"
      }
      ```
    - 错误日志记录
  - 验收标准:
    - 所有异常被捕获并返回标准格式
    - 错误日志包含完整堆栈
    - HTTP状态码正确映射

- [ ] **2.3 错误处理应用**
  - 优先级: P0
  - 预估工时: 2天
  - 文件位置: `src/api/routes/`, `src/services/`
  - 实现内容:
    - 在所有API路由中应用异常处理
    - 在所有服务层添加异常抛出
    - 添加错误恢复逻辑
  - 验收标准:
    - 所有API有try-catch块
    - 错误信息对用户友好
    - 关键错误有恢复机制
